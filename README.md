# Yandex Music Psychograph Bot

Телеграм-бот, который принимает ссылку на **публичный** плейлист Яндекс Музыки, вытягивает список треков и выполняет психографический анализ по Big Five с помощью LLM.

## Docker

- Создайте `.env` на основе [.env.example](./.env.example):
- Продовая схема: сам бот запускается вне РФ, а запросы к Яндекс Музыке идут через SSH/SOCKS-туннель, задаваемый `SSH_ADDR` и `SSH_PORT`.
- Для локального запуска укажите `SSH_KEY_PATH` на существующий приватный ключ, например `/Users/<you>/.ssh/id_rsa`. Для сервера можно оставить `./ssh/id_ru_tunnel`.

```bash
docker compose down -v
docker compose up -d
docker compose logs -f bot
```

## Использование

1. Нажмите `/start`.
2. Выберите кнопку:
   - `Мой музыкальный портрет`
   - `Совместимость с близким`
3. Отправьте ссылку(и) на публичные плейлисты:
```text
https://music.yandex.ru/users/<user>/playlists/<kind>
```

Бот:
- извлечёт треки (исполнитель — трек),
- покажет превью,
- в личном режиме выполнит психографический анализ (Big Five),
- в режиме совместимости посчитает близость вкусов, подсветит общие треки и соберёт совместимый психологический портрет,
- вернёт структурированный ответ.

## Примечания

- Плейлисты должны быть **публичными**. Для приватных используйте `YM_TOKEN` (опционально).
- При очень больших плейлистах анализ может занимать заметное время, особенно в режиме сравнения двух плейлистов.
- Вы можете заменить модель LLM, изменив `MODEL` в `.env`.
- Для продового доступа к Яндекс Музыке нужен рабочий SOCKS-туннель.
- Контейнер сам копирует ключ в `/root/.ssh/id_rsa` с правами `0600`, но `SSH_KEY_PATH` должен указывать на реальный файл на хосте.
